# Tarjeta · Análisis de cobertura

> Tarjeta de 1 página. Un porcentaje no se mira, se lee: el número es la media del país, el informe
> coloreado es el mapa. Sobre **MTP** (`Microsoft.Testing.Extensions.CodeCoverage`), **no Coverlet**.

## Generar el dato

```bash
# cómoda (la de M2.3)
dotnet test tests/VentasShop.TestsUnitarios -- --coverage --coverage-output-format cobertura

# con control (CI / Sonar)
dotnet-coverage collect "dotnet test tests/VentasShop.TestsUnitarios" -f cobertura -o coverage.cobertura.xml
```

- Formato según quién lee: **`cobertura`** → ReportGenerator (y branch coverage completo); **`xml`** → SonarQube.
- Coverlet NO: es de VSTest, no integra con MTP.

## Del XML al mapa

```bash
dotnet tool install --global dotnet-reportgenerator-globaltool
reportgenerator -reports:"**/coverage.cobertura.xml" -targetdir:"coveragereport" -reporttypes:Html
```

## Leer el color

- **Verde**: línea ejecutada por algún test.
- **Rojo**: zona ciega, nadie pasó.
- **Amarillo**: decisión (`if`/`switch`/`Math.Min`) con ramas a medias. El oro: no se ve en ningún %.
- Mira la columna de **ramas**, no la de líneas.

## Qué rojo importa (M2.1: riesgo × consecuencia)

- **Ciudad** (cubrir): lógica de negocio —descuentos, validaciones, transiciones de estado—.
- **Océano** (soltar): getters sin lógica, `ToString()` de depuración, `catch` defensivo imposible.

## SonarQube (tendencia)

- `dotnet-sonarscanner` begin → build → test → end; cobertura por `sonar.cs.vscoveragexml.reportsPaths`
  (XML de VS de `dotnet-coverage -f xml`). Community gratuita. Quality gate sana: «cobertura sobre código nuevo».

## La trampa

- **Verde ≠ probado**: un test sin aserción pinta de verde igual. El mapa dice dónde hay cámara, no que
  enfoque algo. Cazar el verde ciego = mutation testing (M7.2).
