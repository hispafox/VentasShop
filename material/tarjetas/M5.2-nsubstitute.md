# Tarjeta · Mocking con NSubstitute

> Tarjeta de 1 página. Con un doble solo haces dos cosas: le das el guion antes (qué responde) y revisas la
> toma después (cómo se le llamó). En NSubstitute, `Returns` y `Received`.

## Crear el doble

```csharp
var repositorio = Substitute.For<IRepositorioPedidos>();   // sobre la INTERFAZ, no la clase
```

De fábrica no hace nada: cada método devuelve el valor por defecto (`null`, `0`…) y no protesta.

## El guion: `Returns` (stub)

```csharp
repositorio.ObtenerPorId(idPedido).Returns(pedido);
pasarela.Cobrar(Arg.Any<Dinero>(), Arg.Any<string>()).Returns(new ResultadoPago(true));
reloj.Ahora.Returns(new DateTimeOffset(2026, 1, 1, 12, 0, 0, TimeSpan.Zero));
```

- `Arg.Any<T>()` = cualquier valor de ese tipo · `Arg.Is<T>(pred)` = una condición.
- `.Throws(new ...())` para que lance (necesita `using NSubstitute.ExceptionExtensions;`).

## Revisar la toma: `Received` (mock/spy)

```csharp
repositorio.Received(1).Guardar(pedido);                 // se llamó una vez
repositorio.DidNotReceive().Guardar(Arg.Any<Pedido>());  // NO se llamó
repositorio.Received(1).Guardar(Arg.Is<Pedido>(p => p.Estado == EstadoPedido.Pagado)); // con qué
```

Va en el Assert, después del Act. Comprobar que algo **no** pasó vale tanto como que sí.

## Estado vs. interacción

- Prefiere **estado**: `Assert.Equal(EstadoPedido.Pagado, pedido.Estado)`. Más sólido, no se acopla al cómo.
- Usa **interacción** (`Received`) cuando no hay estado que mirar: el rechazo no guarda → `DidNotReceive`.

## Permisivo, no estricto

Un método sin configurar devuelve el valor por defecto y no falla (NSubstitute es *loose*). Cómodo, pero si
tu código esperaba algo y recibe `null`, revienta dentro del SUT con un `NullReferenceException` confuso.

## Si vienes de Moq

| Moq | NSubstitute |
|---|---|
| `new Mock<IRepo>()` | `Substitute.For<IRepo>()` |
| `mock.Setup(x => x.ObtenerPorId(id)).Returns(pedido)` | `repo.ObtenerPorId(id).Returns(pedido)` |
| `It.IsAny<int>()` | `Arg.Any<int>()` |
| `mock.Verify(x => x.Guardar(p), Times.Once)` | `repo.Received(1).Guardar(p)` |
| `mock.Verify(..., Times.Never)` | `repo.DidNotReceive().Guardar(p)` |
| `mock.Object` | el propio `repo` (no existe `.Object`) |

## Errores comunes

- **Verificar de más** → te acopla a la implementación. Verifica solo lo que el test comprueba.
- **Interacción donde bastaba el estado** → mira el estado si puedes.
- **Olvidar el permisivo** → `NullReferenceException` confuso por un método sin configurar.
- **Async** → configura los métodos `Task` que el SUT vaya a esperar.
