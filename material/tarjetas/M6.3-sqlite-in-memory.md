# Tarjeta · Testing con SQLite in-memory

> Tarjeta de 1 página. SQLite in-memory es un **motor relacional de verdad** en RAM: el coche de verdad en
> el circuito de pruebas. Respeta índices únicos, claves foráneas y transacciones. Sin instalar ni levantar nada.

## La clave técnica (que más despista)

```csharp
var conexion = new SqliteConnection("Filename=:memory:");
conexion.Open();   // la base existe MIENTRAS la conexión siga abierta

var opciones = new DbContextOptionsBuilder<ContextoVentasShop>()
    .UseSqlite(conexion)   // el OBJETO conexión, no una cadena de texto
    .Options;
using var contexto = new ContextoVentasShop(opciones);
contexto.Database.EnsureCreated();   // crea el esquema desde el modelo (con el índice único)
```

- Paquete `Microsoft.EntityFrameworkCore.Sqlite`.
- **La base vive mientras la conexión esté abierta** → ábrela y mantenla viva durante el test.
- A `UseSqlite` le pasas el **objeto** conexión, no una cadena (si no, cada operación crea y tira su base).
- **`EnsureCreated`**, no `MigrateAsync` (el repo no tiene migraciones).
- **Una conexión por test** = una base limpia y propia (independencia, M1.3). Constructor abre, `Dispose` cierra.

## Qué SÍ comprueba (lo que el provider in-memory no)

- **Índice único** (dos `Producto.Codigo` iguales → `DbUpdateException` al guardar).
- **Integridad referencial de las claves foráneas** y las transacciones reales.
- **Traducción a SQL** de tus consultas (es SQL de verdad).

## La recompensa (el test que en M6.2 no saltaba)

```csharp
contexto.Productos.Add(ProductoCon("ABC", 50m));
contexto.Productos.Add(ProductoCon("ABC", 20m));   // mismo Codigo → viola el índice único

Action guardar = () => contexto.SaveChanges();
guardar.Should().Throw<DbUpdateException>();        // ¡ahora SÍ salta!
```

## El límite honesto

- SQLite es un motor relacional real, **pero no es tu motor de producción**: hay diferencias de dialecto
  (tipos, funciones, ordenación de cadenas).
- Cubre el grueso de la integración sin montar nada; para el dialecto exacto crítico, tu motor real.

## Errores comunes

- **Dejar que la conexión se cierre** → la base desaparece, «tablas que no existen».
- **Creer que es tu motor de producción** → para el dialecto, el juez es tu motor real.
- **Compartir una conexión entre tests** → comparten base y se pisan; una por test.
- **Olvidar `EnsureCreated`** → la base nace vacía, sin tablas.
