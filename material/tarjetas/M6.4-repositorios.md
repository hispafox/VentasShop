# Tarjeta · Testing de repositorios y acceso a datos

> Tarjeta de 1 página. El patrón Repository es un **mostrador** delante de la base de datos: lo doblas en
> unitario (M5), lo pruebas de verdad en integración (SQLite in-memory, M6.3), y tu lógica queda limpia.

## El repositorio (síncrono, castellano)

```csharp
public interface IRepositorioPedidos
{
    Pedido? ObtenerPorId(Guid id);                       // Find: NO carga navegaciones
    Pedido? ObtenerConLineas(Guid id);                   // con Include de Lineas y Cliente
    IReadOnlyList<Pedido> ObtenerPorCliente(Guid clienteId);
    void Agregar(Pedido pedido);
    void Guardar(Pedido pedido);
    void Eliminar(Pedido pedido);
}
```

## El contraste que solo caza la integración

- `ObtenerPorId` (Find) trae el pedido **sin** sus líneas → si esperas líneas, vienen vacías.
- `ObtenerConLineas` (Include) las trae cargadas.
- Un test con doble nunca lo notaría; contra el motor real, sí. Es la costura del M6.1.

## El CRUD, cada uno un test de integración

- **Crear + leer**: `Agregar`, recuperar en contexto **nuevo** (no la caché de EF, 6.2) con `ObtenerConLineas`.
- **Actualizar**: recuperar, `Confirmar()`, `Guardar`, volver a leer y comprobar el estado.
- **Borrar**: `Eliminar` (cascade a las líneas) y comprobar que ya no está.
- **Consultar**: `ObtenerPorCliente`, comprobar que devuelve los correctos y solo esos.
- Pocos y bien elegidos (M6.1): por comportamiento, no por cada getter trivial.

## El aislamiento entre tests (elige UNA y aplícala a todos)

- **Recrear** el esquema entre tests — simple, lento; suites pequeñas.
- **Limpiar** los datos (Respawn, respeta relaciones) — buen equilibrio.
- **Transacción + rollback** — rápida, pero no pruebas bien tu propio comportamiento transaccional.
- En este repo: **una conexión SQLite in-memory por test** = base nueva cada vez, independencia gratis.

## Errores comunes

- Sin estrategia de aislamiento → fallos según el orden.
- Leer de la caché de EF → falso verde.
- Repositorio con el provider in-memory creyendo que pruebas el acceso a datos → no valida lo relacional.
- Lógica de negocio en el repositorio → va al dominio (unitarios rápidos). El mostrador trae y lleva.
